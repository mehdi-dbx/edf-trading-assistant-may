## What this project is

A weather intelligence chatbot for EDF energy traders. It answers questions about weather forecasts, degree days (HDD/CDD), teleconnection indices (EPO, PNA, NAO, WPO, MJO), and their implications for US power markets (PJM, ISONE, MISO). Built on the Databricks Agent Framework with a LangGraph agent, deployed as a Databricks App.

## Architecture

```
[React Chat UI (Next.js)]  -->  [Node API (port 3001)]  -->  [FastAPI Backend (port 8000)]
                                                                      |
                                                          +-----------+-----------+
                                                          |                       |
                                                  [LangGraph Agent]       [REST endpoints]
                                                          |               /tables, /files
                                                  +-------+-------+
                                                  |               |
                                          [Genie MCP]    [Knowledge Assistants]
                                          (SQL over       (15 KA endpoints for
                                           Delta tables)   document-based Q&A)
```

- **Backend**: FastAPI + MLflow AgentServer (ResponsesAgent interface), LangGraph agent with tool calling
- **Frontend**: React/Vite chat app (`app/`), communicates via Node API proxy
- **LLM**: `databricks-claude-sonnet-4-6` served via Databricks Model Serving
- **Data layer**: Unity Catalog tables in `edf.chatbot` schema, queried via Genie MCP or direct SQL; UC Volumes for document files

- **`tools/query_knowledge_assistant.py`** -- Queries 15 Databricks Knowledge Assistant endpoints by name (daily outlooks, teleconnections, MJO, CWG wind, degree days, ENSO, etc.)
- **`tools/ka_loader.py`** -- Loads KA endpoint mapping from `data/ka.list`

## Deployment

Deploy via Databricks Asset Bundles:
```bash
databricks bundle deploy && databricks bundle run agent_edf_trading_assistant
```

Or use the deploy script:
```bash
./deploy/deploy.sh
```

Permission grants for the app service principal are in `deploy/grant/`:
- `grant_ka_endpoints_for_app.py` -- Grant access to all 15 KA endpoints
- `authorize_genie_for_app.py` -- Grant Genie space access
- `authorize_warehouse_for_app.py` -- Grant SQL warehouse access
- `grant_app_tables.py` -- Grant table access
- `run_all_grants.sh` -- Run all grants at once

Targets: `dev` (development), `prod` (production), `template` (default, maps to `agent-edf-trading-assistant`)

## Known issues / gotchas

- **WeasyPrint on macOS**: Requires `brew install pango cairo` and `DYLD_LIBRARY_PATH=/opt/homebrew/lib` for local PDF generation. Not needed on Databricks.
- **KA endpoint alignment**: Prompt KA names must match `data/ka.list` entries -- use `scripts/check_prompt_ka_names.py` to verify.
- **Genie room**: Set `EDF_TRADING_GENIE_ROOM` env var for Genie MCP to activate; without it, only KA tools are available.
- **OAuth only for deployed app**: Databricks Apps only accept OAuth tokens, not PATs.
